How this feature works

This page is part of the final project feature for IS601.

  • Backend: FastAPI router at /profile using SQLAlchemy to read/update the User table.
  • Security: Passwords are never stored in clear text. All comparisons run against hashed passwords via Passlib bcrypt.
  • Integration tests: Python tests exercise the full flow (load → update → change password) directly against the API.
  • E2E tests: Playwright drives this page in a real browser against http://localhost:8000.
Behind the scenes this feature touches all layers of the stack: database models, Pydantic schemas, FastAPI routes, front-end JavaScript, and automated tests.
